# 真实数据样本生成设计 — 南通大学杏林学院外卖场景

## 目标

为论文验证生成一组真实可信的校园外卖数据样本，模拟南通大学杏林学院学生使用本平台下单、商家接单的完整业务流程。数据将直接写入 MySQL 数据库，用于支撑论文中以下验证点：

1. **多商户隔离**：订单分布在 2 个商家
2. **用户认证与下单**：完整用户-地址-订单链路
3. **订单状态流转**：待接单 → 已接单 → 派送中 → 已完成
4. **经营数据统计**：7 日/30 日营业额趋势、分类占比、热销排行
5. **商家工作台展示**：真实订单列表、待处理提醒

## 数据规模

| 实体 | 数量 | 说明 |
|------|------|------|
| 学生用户 | 8 人 | 南通大学杏林学院在读学生 |
| 收货地址 | 12 条 | 宿舍地址为主，部分含教学楼/图书馆 |
| 订单 | 60 条 | 近 30 天（2026-04-12 ~ 2026-05-11）|
| 订单明细 | ~150 条 | 每单 1~4 个菜品 |

## 技术实现

使用 Python 脚本（`scripts/seed_realistic_data.py`）：

1. 连接本地 MySQL（sky_take_out 数据库）
2. 清空现有测试数据（保留菜品/分类/员工基础数据）
3. 按顺序插入：user → address_book → orders → order_detail
4. 自动处理外键关联（user_id, address_book_id, order_id）
5. 输出数据摘要报告

## 安全与回滚

- 脚本执行前自动备份受影响表
- 提供 `--rollback` 参数可一键还原
- 仅操作 user/address_book/orders/order_detail 四张表，不动菜品和员工数据
