# 校园外卖管理系统 - 前端设计规格书

## 项目概述
**项目名称**: 校园外卖管理系统 (Campus Takeout Admin)  
**类型**: SaaS 管理后台系统  
**目标**: 打造媲美美团、饿了么商家后台的现代化外卖管理界面

---

## 1. 设计系统

### 1.1 色彩方案 (Color Palette)
**主色调**: 温暖橙色系（外卖品牌特征色）
- Primary Orange: `#FF6B35` (主要按钮、强调元素)
- Secondary Orange: `#FFA500` (次级强调)
- Accent Green: `#52C41A` (成功状态、营业中)
- Accent Red: `#FF4D4F` (危险操作、已售罄)

**中性色**:
- Background: `#F5F5F5` (页面背景)
- Card Background: `#FFFFFF` (卡片背景)
- Border: `#E8E8E8` (边框)
- Text Primary: `#262626` (主要文字)
- Text Secondary: `#8C8C8C` (次要文字)

**状态色**:
- Warning: `#FAAD14` (警告)
- Info: `#13C2C2` (信息提示)

**⚠️ 严格禁止**: 蓝色、紫色色系

### 1.2 字体系统 (Typography)
**字体家族**:
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif;
```

**字号比例**:
- Heading 1: 32px / Bold (页面标题)
- Heading 2: 24px / Semi-bold (区块标题)
- Heading 3: 18px / Semi-bold (卡片标题)
- Body Large: 16px / Regular (正文)
- Body: 14px / Regular (默认文字)
- Caption: 12px / Regular (辅助信息)

### 1.3 间距系统 (Spacing)
基于 8px 网格系统:
- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px
- xxl: 48px

### 1.4 圆角系统 (Border Radius)
- Small: 4px (按钮、输入框)
- Medium: 8px (卡片)
- Large: 12px (大卡片、模态框)
- XLarge: 16px (特殊装饰)

### 1.5 阴影系统 (Shadows)
```css
--shadow-sm: 0 1px 2px rgba(0,0,0,0.05);
--shadow-md: 0 4px 6px rgba(0,0,0,0.07);
--shadow-lg: 0 10px 15px rgba(0,0,0,0.1);
--shadow-hover: 0 8px 20px rgba(255,107,53,0.15);
```

## 3. 核心页面设计规格

### 3.1 登录页面 (Login Page)
**视觉风格**: 分屏设计 (50/50)

**左侧**: 品牌展示区
- 背景: 高品质美食摄影（温暖色调，浅景深）
- 元素: Logo、Slogan、装饰性插图
- 提示文字: "现代化外卖管理，从这里开始"

**右侧**: 登录表单区
- 背景: 纯白
- 表单元素:
  - 输入框: 大号（48px 高）、带图标
  - 登录按钮: 全宽、圆角、渐变橙色
  - 记住密码 Checkbox
  - 忘记密码链接
- 动画: 输入框聚焦动画、按钮悬停效果

**响应式**:
- 移动端: 单栏布局，背景图片作为顶部装饰

---

### 3.2 仪表盘页面 (Dashboard)
**布局**: 4列网格系统

**顶部统计卡片** (4张卡片横排):
1. 今日订单量
2. 今日营业额
3. 待处理订单
4. 商品总数

**卡片设计**:
- 尺寸: 每张占 1/4 宽度
- 样式: 白色卡片、渐变色边框装饰
- 内容: 大号数字 + 趋势图标 + 环比数据
- 图标: 使用 SVG 图标（订单、钱袋、闹钟、商品）

**图表区域** (2列布局):
- 左侧 (66% 宽):
  - 标题: "营业趋势"
  - 图表: 7天销售额折线图
  - 交互: 悬停显示详细数据
  
- 右侧 (33% 宽):
  - 标题: "分类销售占比"
  - 图表: 饼图或环形图
  - 显示: 各分类销售百分比

**快捷操作区**:
- 新增商品、查看订单、数据导出等按钮
- 卡片式布局、带图标

**实时订单流**:
- 最新 5 条订单滚动展示
- 状态标签（待接单、配送中等）

---

### 3.3 商品管理页面 (Products)
**顶部操作栏**:
- 搜索框（商品名称搜索）
- 分类筛选下拉菜单
- 状态筛选（全部/在售/售罄）
- 新增商品按钮（主要操作，橙色）

**商品列表** (两种视图切换):
1. **卡片视图** (默认):
   - 4列网格布局
   - 每张卡片:
     - 商品图片 (16:9 比例)
     - 商品名称
     - 价格（大号、橙色）
     - 分类标签
     - 状态开关（上架/下架）
     - 操作按钮（编辑、删除）
   - 悬停效果: 阴影加深、图片轻微放大

2. **表格视图**:
   - 列: 图片、名称、分类、价格、库存、状态、操作
   - 斑马纹背景
   - 行悬停高亮

**商品编辑抽屉** (Drawer):
- 从右侧滑入
- 表单字段:
  - 商品图片上传（支持拖拽）
  - 商品名称、分类、价格、单位
  - 商品描述（富文本编辑器）
  - 规格管理（尺寸、口味等）
  - 状态开关
- 底部: 保存、取消按钮

---

### 3.4 订单管理页面 (Orders)
**状态标签页**:
- 全部订单
- 待接单（红点提示数量）
- 备餐中
- 待配送
- 已完成
- 已取消

**订单列表**:
- 卡片式布局（每个订单一张卡片）
- 卡片内容:
  - 左侧: 订单号、下单时间、顾客信息
  - 中间: 商品列表（图片 + 名称 + 数量）
  - 右侧: 
    - 订单金额（大号）
    - 状态标签
    - 操作按钮（接单、完成、取消）
- 颜色编码:
  - 待接单: 橙色边框
  - 备餐中: 绿色边框
  - 已完成: 灰色

**订单详情模态框**:
- 完整订单信息
- 商品明细表格
- 顾客信息、配送地址
- 备注信息
- 操作日志时间线

**筛选功能**:
- 日期范围选择器
- 订单号搜索
- 金额范围筛选

---

### 3.5 套餐管理页面 (Set Meals)
**设计**: 与商品管理类似，但突出"组合"概念

**套餐卡片**:
- 显示套餐主图
- 套餐名称、价格
- 包含商品数量提示
- 原价、优惠价对比

**套餐编辑**:
- 基本信息（名称、价格、图片）
- 商品选择器（多选、可设置数量）
- 自动计算原价与折扣

---

### 3.6 数据统计页面 (Statistics)
**图表丰富性**:
1. **销售趋势图** (折线图):
   - X轴: 时间
   - Y轴: 销售额/订单量
   - 可切换: 日/周/月视图

2. **分类销售分析** (柱状图):
   - 各分类销售额对比

3. **热销商品排行榜** (表格 + 进度条):
   - Top 10 商品
   - 销售数量可视化

4. **时段分析** (热力图):
   - 每小时订单量分布

**数据导出**:
- 导出按钮
- 支持 CSV、Excel 格式

## 4. 交互动效规范

### 4.1 微动效
- **按钮**: 悬停缩放 1.02、阴影加深
- **卡片**: 悬停上浮 2px、阴影扩散
- **输入框**: 聚焦边框颜色渐变
- **切换开关**: 滑动动画（0.3s ease）
- **抽屉/模态框**: 淡入 + 滑动（0.3s cubic-bezier）

### 4.2 加载状态
- **骨架屏**: 用于内容加载
- **Spinner**: 橙色旋转加载器
- **进度条**: 页面顶部线性进度条

### 4.3 过渡动画
- **页面切换**: 淡入淡出（0.2s）
- **列表项**: 交错动画（stagger effect）

```css
/* Mobile */
@media (max-width: 768px)

/* Tablet */
@media (min-width: 769px) and (max-width: 1024px)

## 6. 技术实现方案

### 6.1 技术栈
- **框架**: 原生 JavaScript (ES6+) + 简易路由
- **样式**: Tailwind CSS (CDN)
- **图表**: Chart.js (轻量级图表库)
- **图标**: Heroicons (SVG 图标)
- **动画**: CSS Transitions + 轻量 JS

### 6.2 架构
- 单页应用 (SPA) 架构
- 组件化设计（模块化 JS）
- 状态管理（简单全局对象）
- 模拟 API（本地 Mock 数据）

### 6.3 文件结构
```
reggie-takeout-admin/
├── index.html                 # 主入口
├── assets/
│   ├── css/
│   │   └── style.css         # 自定义样式
│   ├── js/

│   │   ├── router.js         # 路由管理

## 8. 视觉资源需求

### 8.1 登录页背景
- **描述**: 精美美食摄影，温暖色调，浅景深效果
- **尺寸**: 1920x1080
- **风格**: 专业摄影，高清，温馨氛围
- **主体**: 精致摆盘的中餐菜品，带有蒸汽效果

### 8.2 商品图片（15张）
**分类覆盖**:
- 热菜类: 6张（宫保鸡丁、鱼香肉丝、糖醋里脊等）
- 凉菜类: 3张（拍黄瓜、凉拌木耳等）
- 主食类: 3张（米饭、面条、饺子等）
- 汤类: 3张（番茄蛋花汤、紫菜汤等）

**要求**:
- 高清、专业摆盘
- 统一拍摄角度（45度俯拍）
- 白色或木质餐具背景
- 光线充足、色彩饱和

## 9. 性能优化

### 9.1 图片优化
- 使用 WebP 格式
- 懒加载（Lazy Loading）
- 响应式图片（srcset）

### 9.2 代码优化
- CSS/JS 内联关键资源
- 最小化外部依赖
- 异步加载非关键脚本

- ARIA 标签支持
- 键盘导航支持

## 设计理念总结

> "让外卖管理变得高效而愉悦"

这套系统将结合现代 SaaS 设计语言与外卖行业特性，用温暖的橙色系传达品牌活力，用清晰的信息架构提升操作效率，用流畅的动效增强用户体验。

**核心关键词**: 现代化、高效、温暖、专业
